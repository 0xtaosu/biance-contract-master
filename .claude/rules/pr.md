## PR Format

Each Pull Request must use the following structure:

### Summary
Brief description of what this PR does.

### Related Issue
Closes #<issue-number>

### Changes
- Bullet list of concrete changes

### Tests
- Commands run and results

### Risks
- Potential side effects or risks

### Rollback
- How to revert this change

Rules:
- One PR closes one Issue only.
- Do not include unrelated refactors.
- Do not change architecture unless an ADR is included.
